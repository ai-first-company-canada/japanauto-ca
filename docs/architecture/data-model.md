Per [DOCS-CONVENTIONS.md](../../DOCS-CONVENTIONS.md) R5, verify cited symbols against the code when relying on this document.

## Purpose

Versioned raw-SQL schema for Cloudflare D1 (SQLite) backing japanauto.ca — dealers/salvage-yards, used-car listings, donor-car directory, monetization (featured slots, boosts), media, auth tokens, atomic rate-limit/upload-claim counters, and (0012–0019) the analytics/ops layer: per-day stats rollups with traffic attribution, the private market-stats snapshot, the admin audit trail, and e-mail-report idempotency — with lib/schema.ts (zod) as the application-layer contract that duplicates every DB CHECK/trigger as the API's last line of defence.

## How it works

Storage is Cloudflare D1 (SQLite). Conventions (0001 header): money = INTEGER cents CAD; timestamps = INTEGER unix seconds; entity IDs = TEXT (cuid2/nanoid minted in Workers); reference IDs (makes/models/cities) = INTEGER PK; booleans = INTEGER 0/1; provinces = 13 ISO codes enforced by CHECK on every table that stores one. Migrations are plain numbered .sql applied manually via `wrangler d1 migrations apply japanauto-prod --local|--remote` (no `migrate` npm script; deploy = `wrangler pages deploy`, predeploy gate is typecheck+build+audit:seo only — migrations are NOT part of deploy). \n\nCore entity graph: dealers (id TEXT PK, type dealer|salvage_yard) is the seller root. listings → dealer_id ON DELETE CASCADE, make_id/model_id ON DELETE RESTRICT; donor_cars (added 0005) → dealer_id CASCADE, make_id/model_id RESTRICT, city_slug→cities.slug RESTRICT. makes→models is RESTRICT. cities + city_aliases support edge geolocation: request.cf.city (city-political) → city_aliases(city_political,province PK) → cma_slug → cities; Gatineau aliases deliberately carry province='QC' while mapping to CMA 'ottawa' (ON), so Workers must permit the one cross-province alias. featured_slots (sponsored NEW-vehicle promo, one active per make×city, age-cap-exempt) and boost_orders + listings.boost_until/boost_paid_cents (paid visibility for used listings) implement ADR-0007 monetization. media is polymorphic ((entity_type,entity_id), no DB FK). Auth: refresh_tokens (hashed, rotation chain via self-FK rotated_to ON DELETE SET NULL) + verification_tokens (email_verify|password_reset). contact_reveals logs 'Show contact' clicks with ip_hash=sha256(ip+daily-salt).\n\nRolling 10-year used-car age cap (listings.year): SQLite forbids non-deterministic strftime() in CHECK, so it's a static CHECK (year BETWEEN 1990 AND 2100) PLUS two BEFORE INSERT/UPDATE triggers (trg_listings_age_cap_insert/_update) that RAISE(ABORT) when year < currentYear-10 or > currentYear+1. The window self-shifts each calendar year — no annual migration. lib/schema.ts listingYearWindow()/listingYearSchema duplicate the rule for friendly 422s. donor_cars deliberately has NO rolling cap (static CHECK year BETWEEN 1980 AND 2030) per ADR-0008 — old donors are valuable. featured_slots is cap-exempt by design (advertises model_year >= currentYear).\n\nupdated_at maintenance: AFTER UPDATE triggers per mutable table (dealers/listings/parts→donor_cars/featured_slots) self-set updated_at, guarded by `AND updated_at = OLD.updated_at` to avoid recursion.\n\nzod read/write split: write paths use *CreateInputSchema/*UpdateInputSchema/{table}Schema with full strict rules; read paths use relaxed *RowSchema. listingRowSchema drops the year-window AND VIN ISO-3779 checksum (grey-market JDM imports legitimately fail ISO-3779); dealerRowSchema relaxes phone(E.164) and postal-code format — so a legacy/edge stored value cannot 500 a read of the profile, any joined listing, or the login/refresh path. dealerSelfSchema vs dealerPublicSchema control PII exposure (public drops business_number, gst_number, daily quota counters, password_hash, stripe_customer_id, token_epoch). 0008/0009 counters use single-statement atomicity (INSERT...ON CONFLICT...RETURNING / DELETE...RETURNING) serialized by SQLite's write lock to replace non-atomic KV read-modify-write.

**market_stats (0016, rebuilt 0019)** — daily snapshot of the scraper project's Supabase `japanauto_market_stats` view: per (city_slug, make_slug, model_slug, anchor_year, mileage_bucket, source, seller_kind) it stores n_active + price_p25/p50/p75_cents over ACTIVE marketplace listings and n_delisted + median_days_listed over DELISTED ones, plus computed_on ('YYYY-MM-DD' stamped by the view) and synced_at. mileage_bucket is CHECK-constrained to ('all','0-100k','100-200k','200k+'), seller_kind to ('dealer','private','unknown'). The view emits whole dollars; the sync multiplies by 100 to honour the integer-cents invariant. Written ONLY by `syncMarketStats()` (workers/expire-sweeper/src/index.ts, cron `45 9 * * *`): pages PostgREST, drops rows with unknown bucket/seller_kind (KNOWN_BUCKETS/KNOWN_KINDS sets), then one D1 batch of INSERT OR REPLACE keyed on the full PK followed by `DELETE FROM market_stats WHERE synced_at < ?` to prune rows absent from the fresh snapshot. Read ONLY by GET /api/listings/:id/stats (functions/api/listings/[id]/stats.ts) behind the marketAnalytics entitlement (functions/api/_lib/entitlements.ts, pro/trial), rendered in src/components/sections/StatsModal.astro grouped into source×seller_kind segments. 0019's rebuild backfilled existing rows as seller_kind='unknown' so the cabinet block stays live until the next sync replaces the snapshot.

**admin_audit_log (0017)** — every mutation performed through the admin Worker (workers/admin, admin.japanauto.ca behind Cloudflare Access, decision 0014) lands here: id (UUID), at (unix seconds), admin_email (from the verified Access JWT), action ('slot.create', 'listing.expire', 'dealer.tier_set', 'social.cancel', …), target (entity id/key), details (JSON of inputs, no secrets); idx_admin_audit_at (at DESC) serves the ops page. Writes go through `audit()` (workers/admin/src/lib/audit.ts), which returns false instead of throwing when the row could not be written; callers append `auditMark()`'s loud marker to the flash message so an unaudited mutation is never silent. 0017 also ships **ux_featured_slots_live**: a partial UNIQUE index on featured_slots(city, make_id) WHERE status IN ('pending','active','paused'). The admin panel's clash SELECT is advisory (D1 can't make check-then-act atomic across statements); the index is the real enforcement — workers/admin/src/pages/slots.ts catches the UNIQUE violation on INSERT — and since activation never changes (city, make_id), create-time uniqueness covers the whole lifecycle; 'ended' rows fall out of the partial set so history accumulates freely.

**0018 (decision 0016 — traffic attribution + e-mail reports)** — entity_stats_daily (the 0012 per-day rollup) gains views_social and views_paid (INTEGER NOT NULL DEFAULT 0), bumped atomically inside `recordView()`'s upsert (functions/api/_lib/db.ts) according to `classifyViewSource()`: utm_medium=social AND utm_campaign starting 'boost-' → social (content-factory boosts); utm_medium=catalog_ads AND utm_campaign='pro-promo' → paid (Meta catalog ads, ADR-0015); everything else — including a bare utm_medium someone appended to a shared link — counts as direct/search, derived as views − views_social − views_paid, never stored. dealers.reports_opt_out (INTEGER NOT NULL DEFAULT 0) is the CASL opt-out flag, flipped only by GET /api/reports/unsubscribe (functions/api/reports/unsubscribe.ts) after verifying HMAC-SHA256(REPORTS_UNSUB_SECRET, dealer_id) — no session needed. report_runs (PK (period, dealer_id), dealer_id REFERENCES dealers ON DELETE CASCADE) makes the weekly/monthly report crons idempotent: `sendReports()` (workers/expire-sweeper/src/reports.ts, crons `0 14 * * 1` / `30 14 1 * *`) reserves the row with INSERT OR IGNORE before sending and DELETEs it on send failure, and period keys are anchored to the period itself ('weekly-2026-06-15' = most recent Monday, 'monthly-2026-06'), not the run date, so a re-run on any other day reproduces the same key and dedupes.

## Invariants

- Every province column (dealers, cities, city_aliases, listings, featured_slots) is CHECK-constrained to the 13 ISO codes; PROVINCES in lib/schema.ts must stay identical.
- listings.city and dealers.city both store the city SLUG (kebab), not display name — 0003 backfilled dealers.city to enforce this; new dealer writes must slugify.
- listings.year must be in [currentYear-10, currentYear+1] — enforced by triggers (DB) + listingYearSchema (zod). donor_cars.year is NOT capped (static 1980..2030).
- VIN: listings.vin NOT NULL, CHECK length=17, UNIQUE; full ISO-3779 checksum + char rules enforced ONLY on write (vinSchema). Reads (listingRowSchema) intentionally skip checksum so JDM imports stay readable. donor_cars.vin is nullable.
- Money is always INTEGER cents CAD; price_currency is CHECK = 'CAD' and country CHECK = 'CA' everywhere they appear.
- AMVIC: if dealers.type='dealer' AND province='AB' then amvic_number must be set — enforced ONLY at app layer (amvicRefiner), no DB CHECK. salvage_yard exempt.
- salvage_yard signup requires specializes_in — app layer only (specializesInRefiner), no DB CHECK, so pre-0007 rows stay valid.
- Only salvage_yard dealers may own donor_cars — enforced only at app layer (no DB CHECK / no FK on type).
- media is polymorphic: (entity_type, entity_id) with NO database FK; referential integrity and the 'exactly one is_primary per entity' rule are app-layer responsibilities.
- media.r2_key is globally UNIQUE; entity_type CHECK is ('listing','donor_car','dealer','featured_slot') after 0005 (was 'part').
- One LIVE featured-slot contract per (city, make) — since 0017 enforced at the DB by the partial unique index ux_featured_slots_live (WHERE status IN ('pending','active','paused')); the admin panel's clash SELECT (workers/admin/src/pages/slots.ts) is advisory only. 'ended' rows are exempt, so history accumulates.
- boost_orders.stripe_payment_id is UNIQUE only WHERE NOT NULL (partial unique index) — prevents double-applying the same Stripe charge.
- A listing can be status='active' with a past expires_at between sweeps: sweepExpired() (workers/expire-sweeper/src/index.ts, cron `0 */6 * * *`) flips them every 6 h, but every public-visibility decision MUST still combine status with isListingExpired() (lib/schema.ts) — the sweep narrows the stale window, it does not close it.
- token_epoch (0010) is baked into each 15-min access JWT; requireDealer compares token epoch vs live row — bumping the row epoch invalidates all outstanding access tokens. Access tokens otherwise carry no DB re-check until exp.
- contact_reveals.ip_hash and refresh_tokens IP must be stored hashed (sha256 + daily salt), never raw IP (audit #20 / PII).
- market_stats must NEVER feed a public surface — marketplace asking prices sit systematically below dealer retail (owner decision, Feature 1). Its only read path is GET /api/listings/:id/stats behind the marketAnalytics entitlement (pro/trial); any new consumer must keep it cabinet-only.
- market_stats seller_kind segments (dealer/private/unknown) must never be mixed or summed across each other — they are distinct populations from the scraper contract (2026-06-12); the 0019 PK exists precisely because the 0016 PK let them overwrite each other on sync.
- market_stats money is INTEGER cents like everywhere: the Supabase view emits whole dollars and syncMarketStats() multiplies by 100 at the boundary — never store the view's dollar values raw.
- Every admin-panel mutation must write admin_audit_log via audit() (workers/admin/src/lib/audit.ts); a failed audit write must surface in the flash message (auditMark), never pass silently.
- entity_stats_daily: views_social + views_paid ≤ views per row (recordView() bumps views by 1 and at most one source column); direct/search traffic is DERIVED (views − views_social − views_paid), never stored. Attribution requires our own campaign markers (boost-*/pro-promo) — these splits substantiate the "30% to ads" line shown to paying dealers and must not be spoofable.
- report_runs (period, dealer_id) is the send-idempotency lock: reserve via INSERT OR IGNORE BEFORE sending, DELETE on send failure; period keys are anchored to the period (Monday / calendar month), not the run date.
- dealers.reports_opt_out = 1 excludes the dealer from every report send (CASL); it is flipped only by the HMAC-verified unsubscribe link — no other write path may touch it without equivalent consent semantics.

## Design decisions

- **Raw versioned SQL on D1, no ORM (ADR-0001).** — Integer-cents money, unix-second timestamps, and SQLite-specific constraint/trigger tricks are expressed directly; avoids ORM impedance on edge runtime. *Rejected:* An ORM/query-builder (e.g. Drizzle/Prisma) — rejected to keep the edge bundle thin and retain exact control over CHECK/trigger SQL.
- **Rolling age cap via BEFORE INSERT/UPDATE triggers, not a CHECK.** — SQLite forbids non-deterministic strftime('%Y','now') inside CHECK; triggers let the window self-shift each year with zero annual migration. *Rejected:* (a) Static CHECK with hardcoded years — rejected, needs annual migration. (b) App-layer-only — rejected, triggers are the DB last line of defence behind zod.
- **parts (granular part-level catalog) dropped and replaced by donor_cars (whole-vehicle) in 0005 (ADR-0008).** — Junkyard reality is donor-car level, not SKU level; preconditions verified parts/media/contact_reveals had 0 rows so no data preservation. *Rejected:* Keep granular parts catalog — rejected by ADR-0008 as wrong granularity for salvage yards.
- **Polymorphic media (entity_type,entity_id) with no FK.** — One Cloudflare Images upload pipeline serves listings/donor_cars/dealers/featured_slots without splitting code per entity; app layer owns integrity. *Rejected:* Per-entity tables (listing_media, donor_car_media…) — rejected: would fork the upload pipeline.
- **featured_slots (long-term sponsored NEW-vehicle promo) kept separate from boost_orders + listings.boost_* (one-off paid visibility for USED listings).** — Different lifecycle, age-cap exemption, sort semantics (boost sorts by boost_paid_cents; featured is single-slot), and billing model. *Rejected:* A single featured_until column on listings (the legacy approach, explicitly removed in 0001) — rejected.
- **Rate-limit counter (0008) and media-upload claim (0009) moved into D1 single-statement upserts/deletes.** — KV has no compare-and-swap and is eventually consistent, so concurrent bursts all read a stale count and pass; D1 write-lock serializes INSERT...ON CONFLICT...RETURNING / DELETE...RETURNING, making check-and-mutate atomic. *Rejected:* Keep KV read-modify-write — rejected: only bounded sequential traffic, not parallel brute-force; allowed image_id IDOR.
- **dev-only seed lives in migrations/dev/0006_seed_sample_donor.sql, a sentinel subfolder.** — `wrangler d1 migrations apply --remote` only scans the top-level migrations_dir, so dev seed never reaches prod; numbered prod sequence intentionally skips 0006. *Rejected:* A guarded 0006 in the main dir — rejected: risk of seeding demo data into prod.
- **Separate relaxed read-side zod schemas (listingRowSchema, dealerRowSchema).** — Re-validating stored rows on READ with input-only rules (year window, VIN ISO-3779, E.164 phone, postal format) would 500 legitimate pages (aged listings, JDM VINs, legacy phone) — reads must be permissive, writes strict. *Rejected:* Reuse the strict write schema on reads — rejected (audit #8 e2e 500s).
- **market_stats rebuilt in 0019 (create-copy-drop-rename), not recreated empty.** — SQLite can't extend a PRIMARY KEY in place; the rebuild backfills existing rows as seller_kind='unknown' so the cabinet market block keeps serving data between migration apply and the next 09:45 UTC sync (stale rows are then pruned by synced_at as usual). *Rejected:* DROP + CREATE empty — rejected: the market block would report no_data for up to a day.
- **Featured-slot exclusivity moved into the DB (ux_featured_slots_live, 0017).** — The admin panel's check-then-act SELECT cannot be atomic across D1 statements; a partial unique index over the live statuses is race-proof, and because activation never changes (city, make_id), create-time uniqueness covers the whole lifecycle. *Rejected:* app-layer SELECT enforcement only — rejected by security review 2026-06-12 (concurrent-create race window).
- **View attribution from OUR utm markers only (decision 0016).** — classifyViewSource() requires both the medium AND our campaign marker (utm_campaign boost-* / pro-promo); no referrer sniffing. The splits substantiate a revenue claim ("30% to ads") shown to paying dealers, so they must not be inflatable by anyone appending ?utm_medium=social to a shared link. *Rejected:* referrer-based classification — rejected: unverifiable and trivially spoofable.

## Security notes

- AMVIC requirement, salvage_yard specializes_in requirement, and the salvage_yard→donor_cars ownership rule are app-layer-only (no DB CHECK/FK). A direct DB write or a Workers bug bypasses them — DB will not catch it.
- 'Exactly one is_primary per entity' and all media→owner referential integrity are app-layer-only (polymorphic media has no FK). A bug can orphan media or attach it across owners.
- Cron sweeping lives in a standalone Worker (workers/expire-sweeper — Pages Functions still can't schedule): sweepExpired() flips expired listings and sweeps rate_limits/refresh_tokens/verification_tokens/contact_reveals every 6 h, but pending_media_uploads orphan rows accumulate between manual admin purges (ops.pending_media_purge) — 0009's documented automatic DELETE is not implemented. Visibility correctness still depends on every query calling isListingExpired() — the sweep is best-effort, not the gate.
- httpUrlSchema explicitly restricts to http/https because zod .url() accepts javascript:/data: URIs → stored XSS in <a href> (dealer website, featured_slots.promo_url). This is a deliberate input hardening; any new URL field must reuse httpUrlSchema, not z.url().
- ip storage must be hashed: contact_reveals.ip_hash = sha256(ip + daily-salt via DAILY_IP_HASH_SALT secret); refresh_tokens IP hashed per audit #20. Raw IP must never be persisted.
- token_epoch (0010) is the only revocation for 15-min access JWTs — without bumping it, logout/password-change do not invalidate an already-issued access token (it snapshots verified/dealer_type and is trusted until exp). Any new revocation event (suspension, password-reset confirm) must bump token_epoch.
- pending_media_uploads (0009) closes an IDOR (finding #14): finalize must consume the dealer-bound claim atomically; without it a dealer could finalize another dealer's public CF image_id onto their own listing.
- dealerPublicSchema must drop business_number/gst_number (CRA/GST tax IDs) — they are never rendered and must not leak via cacheable JSON; phone/email are intentionally public (direct-contact model). Any new public dealer endpoint must use dealerPublicSchema, not dealerSelfSchema/dealerRowSchema.
- 0005 runs with PRAGMA foreign_keys=OFF during the media/contact_reveals rebuild; correctness depends on it restoring foreign_keys=ON at the end (it does) — a partial/failed apply could leave FKs disabled.
- The unsubscribe flip of dealers.reports_opt_out is authorized solely by a timing-safe HMAC check (`hmacHex(secret, "reports-unsub:v1:" + dealerId)` in functions/api/reports/unsubscribe.ts); missing REPORTS_UNSUB_SECRET → 503, fail closed. The secret therefore guards a dealer-data write — treat it like an auth secret, and any new signed-link feature must version its message prefix the same way.
- admin_audit_log.details is a JSON blob of action inputs and is constrained by convention only ('no secrets', 0017 header) — there is no redaction layer; any new admin action must keep secrets/PII out of the details it passes to audit().
