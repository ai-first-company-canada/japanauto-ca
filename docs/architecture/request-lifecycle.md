# Request lifecycle & middleware
> Captured 2026-06-11 from a code-level read of the repository. Per [DOCS-CONVENTIONS.md](../../DOCS-CONVENTIONS.md) R5, verify cited symbols against the code when relying on this document.

## Purpose

A single Cloudflare Pages Functions root middleware (functions/_middleware.ts) that runs on every request before route handlers and static-asset serving. It enforces edge-level security (preview de-indexing, CSRF, body-size cap, /dealer/* auth guard, security headers + CSP), tags bots for analytics gating, mints a per-request CSP nonce for dynamic HTML, handles /api/* CORS, and wraps next() in an error boundary that preserves the JSON ApiError contract.

## Key files

| Path | Role |
|---|---|
| `functions/_middleware.ts` | Root Pages middleware; auto-discovered by Pages at functions/ root. Owns the entire pre-handler request pipeline and post-handler header application. |
| `functions/api/_lib/csrf.ts` | isAllowedOrigin() + isCrossSiteUnsafe(): Sec-Fetch-Site/Origin-based CSRF and CORS origin allowlist logic. Layer-1 enforcement is in the middleware. |
| `functions/_lib/csp-script-hashes.ts` | AUTO-GENERATED SHA-256 hash list (SCRIPT_HASHES) of every unique executable inline <script> body in dist/. Imported by middleware to build script-src without 'unsafe-inline'. Committed so typecheck works without a build. |
| `scripts/generate-csp-hashes.mjs` | Postbuild generator for csp-script-hashes.ts; also a build gate that FAILS the build on inline on*= handlers (>0), on empty dist, and on >64 unique hashes (MAX_HASHES). |
| `functions/_lib/page-shell.ts` | takeCspNonce(data): dynamic HTML routes pull the middleware-minted nonce and set data.cspNonceUsed=true so the nonce source is only emitted when actually consumed. renderShell() injects nonce into inline <script> tags. |
| `functions/api/_lib/auth.ts` | verifyAccessToken() (HS256, alg-pinned, fail-closed on bad JWT_SECRET) and requireDealer() (layer-2 CSRF re-check for cookie-auth, only when no Bearer). AccessTokenPayload carries sub + token_epoch. |
| `functions/api/_lib/db.ts` | getDealerById() — used by middleware session kill-switch to compare live dealers.token_epoch against the token claim. |
| `functions/api/_lib/geolocation.ts` | CityResolution interface (MiddlewareData.geo type). Resolution is no longer called in the hot path (audit #24). |
| `functions/api/_lib/response.ts` | forbidden() => jsonError(403,'forbidden') envelope used by the CSRF rejection. |

## How it works

onRequest (functions/_middleware.ts) destructures {request, env, next, data}, casts data to MiddlewareData (geo?, isBot?, dealerId?, cspNonce?, cspNonceUsed?), parses URL. Pipeline order (early-return gates first): (0) Preview-host bot guard: if hostname matches /\.pages\.dev$/i AND path is /robots.txt, return a hardcoded Disallow:/ robots body with x-robots-tag noindex,nofollow, 200, cache 300s — keeps *.pages.dev out of indexes vs canonical japanauto.ca. (0b) CSRF: isApi = path startsWith /api/; if isApi && isCrossSiteUnsafe(request,env) → forbidden('Cross-site request rejected') BEFORE any handler/D1. isCrossSiteUnsafe: safe methods pass; Sec-Fetch-Site same-origin|none passes; else judge by Origin (must equal own origin or be in isAllowedOrigin allowlist); no Origin → allow only if SFS absent (non-browser client like curl/Stripe webhook). (Body-size) isApi && POST/PUT/PATCH with content-length >64KB → 413 payload_too_large JSON. (1) City resolution is intentionally REMOVED from the hot path (audit #24) — comment only; MiddlewareData.geo retained for a future lazy consumer but nothing reads it today. (2) Bot tag: md.isBot = BOT_UA_RE.test(UA) — UA regex over googlebot/bingbot/gptbot/claude/perplexity/etc.; analytics-gating only, never blocks. (2a) If !isApi, mint md.cspNonce = base64 of 16 crypto.getRandomValues bytes. (2b) /dealer/* auth guard via isDealerProtected (exempts /dealer/login|signup|logout|forgot-password and prefix /dealer/reset-password/, /dealer/verify-email/): read jc_access cookie via regex; missing → 302 to /dealer/login/?next=<encoded path>; verifyAccessToken (signature+exp+iss+type) fails → 302; then session kill-switch: getDealerById(payload.sub) and reject if missing or dealer.token_epoch !== payload.token_epoch (audit #11); on success md.dealerId = payload.sub. (3) CORS preflight: isApi && OPTIONS → if isAllowedOrigin(origin) return 204 with ACAO=origin, allow-methods GET/POST/PATCH/DELETE/OPTIONS, allow-headers content-type,authorization, allow-credentials true, max-age 86400, Vary:Origin; else 403. (4) Error boundary: response = await next() in try/catch (audit #23) — db.ts read helpers call zod .parse() which throws on D1/schema drift; catch logs and returns 500 internal_error JSON so the response still gets headered. (5) Header application (runs for ALL responses, API and HTML/static): lift Set-Cookie via getSetCookie() and re-append individually (Workers Headers iteration collapses multiple Set-Cookie into one comma-joined value), set the 5 SECURITY_HEADERS, set content-security-policy = buildCsp(md.cspNonce) only when md.cspNonceUsed && md.cspNonce, else the precomputed CSP_STATIC (nonce-free, computed once per isolate). If isApi and origin allowed, add ACAO + allow-credentials + Vary:Origin to the actual response. (6) Edge HTML rewriting removed 2026-05-19 (city now baked at SSG/render time). Returns new Response(response.body, {status, statusText, headers}). buildCsp composes default-src 'self'; img-src adds imagedelivery.net; script-src 'self' + optional 'nonce-…' + SCRIPT_HASHES + js.stripe.com (NO 'unsafe-inline'); style-src keeps 'unsafe-inline' + fonts.googleapis.com; font-src fonts.gstatic.com; connect-src api.stripe.com; frame-src Stripe; object-src 'none'; base-uri/form-action/frame-ancestors 'self'.

## Invariants

- script-src MUST NOT contain 'unsafe-inline' or 'unsafe-hashes'. Inline scripts are allowlisted only by build-time SHA-256 hash (SSG pages) or per-request nonce (dynamic page-shell routes); inline event-handler attributes (on*=) are allowed by neither and the build (generate-csp-hashes.mjs) fails if any ship.
- The per-request CSP nonce source is emitted in script-src ONLY when md.cspNonceUsed is true (a route called takeCspNonce). Cacheable static/SSG responses must never advertise an unused nonce, so CSP_STATIC is served for them.
- SECURITY_HEADERS and a CSP header are applied to EVERY response (API, dynamic HTML, and static assets) after next(), including the synthesized 500 error envelope — no response leaves the edge unheadered.
- Multiple Set-Cookie headers must survive header re-construction: they are lifted via getSetCookie() and re-appended individually because new Headers(response.headers) collapses them into one comma-joined value.
- CSRF check runs before any handler/D1 work for all /api/* unsafe methods; requireDealer() re-checks (auth.ts:248) as a second layer but only for cookie-auth requests with no Bearer token.
- Dealer session validity = signature+exp+iss+type valid AND dealers.token_epoch matches the token's token_epoch claim. A signed-but-stale token (post logout/reset/suspend, epoch bumped) is rejected at the /dealer/* guard and in requireDealer.
- JWT verification pins alg=HS256/typ=JWT from a trusted constant, never from the token header (alg-confusion/alg:none defense), and fails closed (returns null → 401/redirect) on a misconfigured JWT_SECRET rather than 500 or verifying against an empty key.
- Bot detection (md.isBot) is advisory for analytics gating only and must never gate access control.
- Pages preview hosts (*.pages.dev) must serve a Disallow:/ robots.txt + x-robots-tag noindex to avoid duplicate-content penalties; canonical is https://japanauto.ca.

## Design decisions

- **Drop live city/geo resolution from the middleware hot path (step 1 is comment-only); keep MiddlewareData.geo as a dormant field.** — City-first URL architecture bakes city into every page at SSG/render time, so no handler reads context.data.geo. Resolving it cost an uncached D1 alias lookup + KV read on every HTML request for a value that was immediately discarded (audit #24). *Rejected:* Keep resolving geo on every request (rejected: pure per-request D1+KV cost for an unused value).
- **Allowlist inline scripts via build-time SHA-256 hashes (SSG) plus a per-request nonce (dynamic routes) sharing one CSP header, instead of 'unsafe-inline'.** — Lets script-src drop 'unsafe-inline' entirely while supporting ~24 unique inline bodies across 900+ SSG pages and runtime page-shell HTML (audit #18). Hash list is committed so typecheck runs without a build and regenerates each build. *Rejected:* 'unsafe-inline' (rejected: defeats CSP XSS protection); 'unsafe-hashes' for handlers (rejected: weakens policy and was deliberately excluded so the build gate catches on*= regressions); nonce-only (rejected: SSG output is static/cacheable, can't carry a per-request nonce).
- **Emit the nonce source in script-src only after a route consumes the nonce (takeCspNonce sets cspNonceUsed).** — Static/SSG responses are cacheable and must not advertise a one-time nonce that would be reused across cache hits; precomputed CSP_STATIC covers them with zero per-request work. *Rejected:* Always include a nonce source (rejected: nonce reuse across cached responses defeats the nonce's purpose).
- **Wrap next() in try/catch and synthesize a 500 JSON envelope.** — db.ts read helpers use zod .parse() (throws, not safeParse) on raw D1 rows; schema/data drift would throw out of next() and the platform 500 would carry none of the SECURITY_HEADERS/CORS and break the JSON ApiError contract (audit #23). *Rejected:* Let exceptions propagate to the platform default 500 (rejected: unheadered, non-JSON, breaks API contract).
- **Two-layer CSRF: enforce in middleware for all /api/* unsafe methods AND re-check in requireDealer.** — Middleware covers cookie-auth endpoints that bypass requireDealer (/api/auth/refresh, /api/auth/logout); requireDealer re-checks in case the middleware is ever bypassed or refactored (audit #9, defense in depth). *Rejected:* Single check in requireDealer only (rejected: misses cookie-authed refresh/logout); single check in middleware only (rejected: no redundancy if middleware order changes).
- **Reject Origin-less unsafe requests as non-browser (allow) rather than block.** — curl/Stripe webhook deliveries carry no victim cookies, so CSRF cannot apply; blocking them would break legitimate server-to-server calls. *Rejected:* Block all requests lacking Origin (rejected: breaks Stripe webhooks and CLI clients).
- **Keep 'unsafe-inline' in style-src.** — Astro inlines all stylesheets (inlineStylesheets:'always') and markup uses style="" attributes; hashing those is deferred follow-up. *Rejected:* Hash/nonce styles now (deferred, tracked as follow-up).

## Security notes

- CSP script-src has no 'unsafe-inline'/'unsafe-hashes'; XSS via injected inline scripts/handlers is mitigated by hash+nonce allowlisting. style-src still allows 'unsafe-inline' (known residual; CSS-injection/exfil surface remains).
- CSRF defended at the edge for all /api/* unsafe methods via Sec-Fetch-Site/Origin (csrf.ts), pre-empting any D1 work; second layer in requireDealer.
- 64KB /api/* write body cap (POST/PUT/PATCH) bounds storage-amplification via large JSON; uses content-length header — a client omitting/lying about content-length is not pre-checked here (handlers must still bound parsed input).
- JWT verification is alg-pinned to HS256 and fails closed on bad JWT_SECRET (defends alg-confusion/alg:none, avoids verifying against empty key).
- Session kill-switch via dealers.token_epoch invalidates signed-but-unexpired tokens after logout/reset/suspend (audit #11) — but costs a getDealerById D1 read on every protected /dealer/* request.
- HSTS preload (max-age 31536000; includeSubDomains; preload), X-Frame-Options SAMEORIGIN + frame-ancestors 'self', X-Content-Type-Options nosniff, strict-origin-when-cross-origin referrer policy, and a restrictive Permissions-Policy are set on every response.
- CORS allows credentials with a reflected (allowlisted) Origin only; preview *.pages.dev origins are accepted only when env.ENV !== 'production', localhost only when env.ENV === 'dev'.
- jc_access cookie is parsed with a hand-rolled regex /(?:^|;\s*)jc_access=([^;]+)/; HttpOnly/SameSite=Lax cookies are the auth transport (per csrf.ts comment).
- Bot detection is UA-string based and trivially spoofable — correctly used only for analytics gating, never access control.
- Preview *.pages.dev hosts are de-indexed via a synthetic robots.txt + x-robots-tag; relies on the host regex /\.pages\.dev$/i and does not block content access, only crawling.

## Gaps / TODO

- Step 1 (city/geo resolution) is a no-op comment; MiddlewareData.geo is a dormant field with no live producer or consumer (only a stale functions/README.md reference at line 141). Dead-ish surface kept 'for any future lazy consumer'.
- style-src still carries 'unsafe-inline'; hashing/noncing Astro's inlined stylesheets and style="" attributes is an open follow-up explicitly noted in the CSP comment.
- Body-size guard trusts the content-length header only; a chunked/transfer-encoding or absent content-length request bypasses the 64KB pre-check (Number.isFinite guard means missing length parses to 0 and passes).
- csp-script-hashes.ts is committed and only regenerates on `npm run build`; if a developer edits inline scripts and runs typecheck/deploy without a fresh build, the hash list can drift from dist/ (mitigated by deploy gating on build per project memory, not by the middleware itself).
- Edge HTML rewriting and data-geo markers are removed (2026-05-19); step 6 is comment-only. Any residual data-geo* markers in templates fall back to static SSG values with no runtime substitution.
- No rate limiting in the middleware itself; throttling (e.g. login) lives in handlers, so the middleware offers no global request-rate backstop.
